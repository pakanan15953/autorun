## 📐 โครงสร้างการทำงานของบอท (How it works)

โปรเจกต์นี้ใช้เทคนิคหลัก ๆ 4 ส่วนในการแปลงระบบวิเคราะห์ภาพให้กลายเป็นบอทอัจฉริยะ:

### 1. ตา (Screen Capture via `mss`)
เราใช้ไลบรารี `mss` แทน `pyautogui.screenshot()` เนื่องจาก `mss` ดึงภาพหน้าจอได้เร็วมาก (สูงถึง 60+ FPS) เหมาะสำหรับเกมแนววิ่งเร็วอย่าง Cookie Run ที่ต้องอาศัยการประมวลผลเฟรมต่อเฟรมที่แม่นยำ

### 2. สมอง (OpenCV + Dual-Zone ROI)
บอทของเราจะแบ่งพื้นที่หน้าจอ (Region of Interest - ROI) ออกเป็น 2 ส่วนหลัก ๆ:
*   **โซนครึ่งล่าง (Bottom Zone - สีฟ้า):** สแกนหาสิ่งกีดขวางระดับพื้นดิน หรือเหวที่ต้องกระโดด
*   **โซนครึ่งบน (Top Zone - สีเหลือง):** สแกนหาสิ่งกีดขวางระดับหัว หรือป้ายที่ลอยอยู่กลางอากาศที่ต้องสไลด์/หมอบ

เราใช้ **Canny Edge Detection** (การตรวจจับเส้นขอบวัตถุ) แทน Template Matching แบบรูปภาพ เพราะ:
*   **ทนทานต่อฉากหลังเปลี่ยน:** แม้สีด่านจะเปลี่ยน หรือแสงไฟจะเปลี่ยน Canny ก็ยังจับขอบของสิ่งกีดขวางได้
*   **ไม่ต้องเก็บรูปภาพตัวอย่างจำนวนมาก:** ลดความยุ่งยากในการแคปรูปสิ่งกีดขวางทุกอันในเกม

### 3. ระบบส่งคำสั่งแบบ DirectInput (Hardware-level Keypress)
เกมและ Emulator ส่วนใหญ่มีระบบป้องกันระดับหนึ่ง ซึ่งหากใช้คำสั่งพื้นฐานอย่าง `pyautogui.press()` ปุ่มมักจะไม่ทำงานในเกม บอทตัวนี้จึงใช้ **Windows API (`ctypes.windll.user32.SendInput`)** เพื่อจำลองการกดแป้นพิมพ์ส่งตรงไปยังไดรเวอร์เสมือน เสมือนว่ามีคนกดปุ่มจริง ๆ บนบอร์ด

*   **ปุ่ม Jump (กระโดด):** ตั้งค่าเริ่มต้นไว้ที่ปุ่ม `Z` (Scan code: `0x2C`)
*   **ปุ่ม Slide (หมอบ):** ตั้งค่าเริ่มต้นไว้ที่ปุ่ม `X` (Scan code: `0x2D`)
*(คุณต้องเข้าไปที่ Settings ของ Emulator แล้วตั้งค่าปุ่มสำหรับควบคุมเกมให้ตรงกับบอท)*

### 4. การสุ่มพฤติกรรมเลียนแบบมนุษย์ (Anti-Cheat Randomization)
เพื่อป้องกันไม่ให้ระบบหลังบ้านตรวจพบรูปแบบการทำงานซ้ำ ๆ (Pattern Detection) บอทจะใช้เทคนิค **Humanization** ดังนี้:
1.  **สุ่มหน่วงเวลาสายตา (Reaction Delay):** เมื่อเห็นสิ่งกีดขวาง จะสุ่มเวลารอ 10–40 มิลลิวินาทีก่อนจะกดปุ่ม (เพราะสายตามนุษย์ไม่มีทางส่งข้อมูลถึงสมองแล้วกดปุ่มใน 0.00 วินาที)
2.  **สุ่มเวลากดค้าง (Hold Duration):**
    *   การกระโดด: สุ่มกดค้างไว้ 60–110 มิลลิวินาที
    *   การหมอบ: สุ่มกดค้างไว้ 450–650 มิลลิวินาที (ขึ้นอยู่กับระยะสิ่งกีดขวางลอยฟ้า)
3.  **สุ่มช่วงเวลาหน่วงการตอบสนอง (Cooldown Randomization):** เพื่อไม่ให้บอทกระโดดหรือหมอบติด ๆ กันเป็นจังหวะวิเคราะห์ที่เป๊ะเกินไป

## 🔬 การตรวจจับสถานะของเกม (Game State Detection) - ข้อมูลเพิ่มเติม

หากต้องการพัฒนาให้บอทมีความฉลาดมากขึ้นจนสามารถ **กดเริ่มเกมใหม่เองได้** หรือ **หยุดทำงานเมื่อเกมโอเวอร์** สามารถเขียนลอจิกเพิ่มเข้าไปได้ดังนี้ครับ:

### วิธีที่ 1: ตรวจจับสีพิกเซลเฉพาะจุด (Pixel Color Check)
เป็นวิธีที่เบาเครื่องและรวดเร็วที่สุด:
1.  หาตำแหน่งปุ่ม "เริ่มเกม (Play)" หรือปุ่ม "เริ่มใหม่อีกครั้ง (Retry)" ในหน้าต่าง Emulator
2.  อ่านค่าสี RGB ณ พิกเซลตรงนั้น (เช่น ถ้าเป็นปุ่มเริ่มเกมมักจะเป็นสีเขียวหรือเหลืองสดที่เป็นเอกลักษณ์)
3.  เขียนโค้ดเช็ค:
    ```python
    # เช็คว่าอยู่ในหน้า Lobby หรือหน้าสรุปผลการวิ่ง
    # สมมติ พิกเซลที่ X=400, Y=300 ต้องเป็นสีเหลืองของปุ่ม Play (RGB: 255, 215, 0)
    pixel_color = frame[300, 400]
    if pixel_color[2] > 240 and pixel_color[1] > 200:  # เช็คค่า R และ G
        print("เจอหน้า Lobby! สั่งคลิกเพื่อเริ่มเกม")
        # จำลองการคลิกเมาส์
    ```

### วิธีที่ 2: Template Matching เฉพาะจุด (Template Matching for Buttons)
เหมาะสำหรับตรวจจับไอคอนที่ซับซ้อน เช่น ปุ่มรูปหัวใจ หรือป้าย "Game Over"
1.  แคปรูปปุ่ม "Play Again" หรือคำว่า "Game Over" เซฟเป็นรูปเล็กๆ เช่น `retry_btn.png`
2.  สแกนหาในบริเวณมุมล่างหรือจุดกลางจอโดยเฉพาะ (จำกัดพื้นที่ตรวจจับเพื่อไม่ให้หน่วง FPS)
3.  ถ้าค่าความแม่นยำสูงกว่า threshold (เช่น `> 0.85`) ให้สั่งคลิกที่พิกเซลนั้น และเปลี่ยนสถานะบอทเป็น "STARTING"

## 🎨 วิธีทดสอบและจูนกล่องประมวลผล (Tuning Guide)

เมื่อรันโค้ด `python bot.py` จะมีหน้าต่างปรากฏขึ้นมา 2 จอ:
1.  **Bot Vision (Original):** จะเห็นภาพเกมสดๆ พร้อมกรอบกล่องสีเหลือง (บน) และสีฟ้า (ล่าง) เพื่อบ่งบอกโซนสแกน
2.  **Bot Vision (Canny Edges Only):** จะเป็นจอขาวดำ แสดงเฉพาะขอบเส้นที่ OpenCV ตรวจจับได้

### สิ่งที่คุณต้องปรับเปลี่ยนในโค้ดให้เข้ากับจอคอมพิวเตอร์ของคุณ:
*   **พิกัดหน้าจอเกม (`monitor`):** ปรับพิกัด `top`, `left`, `width`, `height` ให้กรอบครอบคลุม Emulator ของคุณพอดี
*   **ตำแหน่งตัวละครและกล่องสแกน (`roi_x_start`, `roi_x_end`):** ปรับให้กล่องสแกนอยู่ด้านหน้าตัวละครพอสมควร หากบอทกระโดดช้าไป ให้ขยับกล่องไปทางขวาเพิ่มขึ้น หรือหากกระโดดเร็วไป ให้ขยับกล่องกลับมาทางซ้าย
*   **พิกเซลขอบตรวจจับ (`count_bottom > 200` และ `count_top > 150`):** ให้สังเกตค่าพิกเซลบนหน้าจอขณะเล่นปกติว่าสิ่งกีดขวางเข้ามามีจำนวนพิกเซลเฉลี่ยเท่าไร แล้วปรับให้ค่านี้เหมาะสมกับขนาดของภาพ
